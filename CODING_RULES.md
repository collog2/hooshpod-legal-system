# Project Rules

- Do not skip planning.
- Do not generate the entire project at once.
- Never duplicate code between Cases, Contracts and Notices.
- Use generic Document, Deadline, Task and ActivityLog entities.
- Never bypass RBAC or ownership checks.
- All database access goes through Prisma.
- Use DTOs and class-validator.
- Use feature modules.
- Do not introduce new libraries without approval.
- Follow NestJS best practices.
- Follow Vue Composition API.
- Prefer reusable components over duplication.
- Update README when setup changes.
